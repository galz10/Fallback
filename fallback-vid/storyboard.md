# Fallback Launch Video Storyboard

Target length: 1:15 to 1:25.

This version is updated around the five clean manual recordings now in `demos/raw/`: watched repos, My Work, PR review, Local Changes, and Command Palette/GitHub Status. Branch Watch remains a strong product message, but it should be treated as a brief graphic/still unless a new manual clip is recorded.

## 0:00 - 0:02 | Logo Reveal

Visuals and animations: On a black background, the Fallback mark draws itself like a branch graph. First the upper horizontal branch, then the vertical branch, then the curved lower branch. The final light node pops in with a subtle glow. The word `Fallback` fades in beside it.

On-screen text:

> Fallback

Asset source:

- `assets/fallback-mark-light.png`
- `assets/fallback-logo-light.png`

## 0:03 - 0:07 | Core Concept

Visuals and animations: Large centered type appears. The words `PRs`, `issues`, `reviews`, `checks`, and `local changes` appear as small chips, then resolve into one line. Keep the motion calm and precise.

On-screen text:

> GitHub context, saved where you work.

Secondary:

> Connected to the local repo state in front of you.

Optional role chips:

- Maintainer.
- Reviewer.
- Release lead.
- Staff engineer.

## 0:08 - 0:18 | Watched Repositories

Visuals and animations: Cut to the Repositories view. Show watched repository cards, cache/storage summary, freshness, repo health, local changes, PR counts, and issue counts. Let the cursor search/filter the repo list. Use a slow crop-in from the cache bar to the repo cards.

On-screen text:

> Watch repos. Keep the working memory local.

Demo source:

- `demos/raw/watch-repo.mp4`

UI beats:

- `Repositories`
- `3 watched repositories`
- storage/cache summary
- cached PR and issue counts
- `Fresh`, `Clean`, `1 file changed`

## 0:19 - 0:32 | My Work Queue

Visuals and animations: The sidebar cursor selects My Work. Show Needs me, Waiting, At risk, Snoozed, Muted, and Done lanes. Feature requested reviews, mentions, and lower-priority grouping. Add subtle labels over the groups rather than heavy callouts.

On-screen text:

> Your cross-repo attention queue.

Secondary overlay:

> Reviews, mentions, risk, drafts, and follow-ups.

Demo source:

- `demos/raw/work-queue.mp4`

## 0:33 - 0:50 | Pull Request Review

Visuals and animations: Move from My Work into a PR. Show the activity/conversation surface, then the Changes tab with file tree, changed-file counts, and syntax-highlighted diff. Use this as the main "GitHub context is local and inspectable" proof point.

On-screen text:

> Review GitHub work without losing the thread.

Secondary overlay:

> Conversations, files, diffs, and review context in one place.

Demo source:

- `demos/raw/pull-request-flow.mp4`

## 0:51 - 1:00 | Local Changes Safety

Visuals and animations: Cut to Local Changes. Show the selected repo branch, changed file count, stash/commit controls, file list, and the inline red/green diff. Highlight that this is real working-tree state, not a static dashboard.

On-screen text:

> Risky Git work gets a local trail.

Secondary overlay:

> Inspect before you stage, stash, commit, or recover.

Demo source:

- `demos/raw/local-changes.mp4`

## 1:01 - 1:10 | Command Palette And Status

Visuals and animations: Open the command palette, search for `open`, then navigate into GitHub Status. Let the palette feel like the app's power-user layer. Finish on service status and uptime rows to underline trust and operational awareness.

On-screen text:

> Keep working when GitHub is having a moment.

Secondary overlay:

> Fast navigation. Clear health signals.

Demo source:

- `demos/raw/command-palette-github-status.mp4`

## 1:11 - 1:17 | Branch Watch Graphic Beat

Visuals and animations: Use a short animated graphic or one of the Branch Watch stills. Draw a branch line, flash a suspicious commit, then reveal evidence labels: landed, expected, tested, recovery. Keep this quick unless a new Branch Watch video pickup is recorded.

On-screen text:

> Suspicious branch changes, explained with evidence.

Reference source:

- `demos/raw/branch-watch-hero.png`

## 1:18 - 1:23 | Local Workflow Ecosystem

Visuals and animations: Return to a clean motion graphic around the Fallback mark. Orbit only true ecosystem pieces: GitHub, Git, local repo, Finder, Terminal, editor, OS keychain, and GitHub Actions. Keep this framed as the repo workflow Fallback organizes, not as a broad integration platform.

On-screen text:

> Built for repo work that needs proof before action.

Important wording: Do not show AI agent avatars or agent collaboration language. Fallback does not currently have AI integrations.

## 1:24 - 1:28 | Outro And CTA

Visuals and animations: Fade to the Fallback logo on black. Small platform badges appear below for macOS, Windows, and Linux. The final node in the logo pulses once.

On-screen text:

> Try the Fallback public alpha.

Secondary:

> Available for macOS, Windows, and Linux.
