# Manage Repositories

Fallback works best when it watches the repositories that matter to your daily work and ignores the ones that do not.

## Add A Repository

Use the repository picker or command palette to add a repository. You can search available repositories from your connected GitHub account or paste `owner/name` directly.

After adding the repository, Fallback starts syncing its GitHub context.

## Choose Metadata-Only Or Cloned

Use metadata-only mode for repositories you mostly read.

Use cloned mode for repositories where you want local code and Git features.

You can set defaults in Settings so new repositories use the mode you prefer.

## Refresh A Repository

Refreshing asks Fallback to sync the latest GitHub context for a repository.

Refresh when:

- You are about to review or comment.
- A pull request changed recently.
- Checks look stale.
- You are about to go offline and want a fresh local snapshot.

## Open A Repository Elsewhere

For cloned repositories, Fallback can open the local folder in:

- Your editor.
- Your terminal.
- Finder or the file browser.

For any watched repository with a GitHub URL, Fallback can open the repository, issues, or pull requests on GitHub.

## Group Repositories

Use repository groups when your watched list grows. Groups help you filter by team, product, client, or workstream.

Groups are only for organization inside Fallback. They do not change GitHub permissions or repository settings.

## Remove A Repository

If a repository no longer matters, unwatch it. Read the confirmation carefully so you understand whether you are only removing it from Fallback or also deleting local cached data or managed folders.

Unwatching does not delete the repository on GitHub.
