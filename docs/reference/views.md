# Views

Fallback is organized into focused views. Each view answers a different kind of question.

## Home

Home is the starting point. Use it to connect GitHub, choose repositories, and orient yourself when the app has no current repository selected.

## My Work

My Work is the daily cross-repo queue. Each item explains why it is present, what changed, and the next likely action.

It can include review requests, assigned issues, failing checks, direct mentions, stale or resumable review drafts, blocked writebacks, snoozed work, muted threads, and work where you are already involved.

The main lanes are Needs me, Waiting, At risk, Snoozed, Muted, and Done.

## Code

Code is the repository workspace view.

Use it to:

- Read the README.
- Browse files.
- Open files.
- Search commits.
- Inspect commit history and commit graph information.
- Switch branches.
- Manage Parallel Workspaces.
- Fetch, pull, push, or publish when a local folder is available.
- Open the repository in an editor, terminal, Finder, or GitHub.

## Local Changes

Local Changes shows changes in a cloned repository's working tree.

Use it to:

- Review changed files.
- Inspect diffs.
- Stage and unstage files.
- Commit changes.
- Stash changes.
- Apply, pop, or drop stashes.
- Discard unwanted file changes.
- Resolve conflict state.
- View file history or blame.

## Pull Requests

Pull requests shows cached pull requests for the selected repository.

Use it to:

- Search and filter pull requests.
- Read conversations.
- Review diffs.
- Save review drafts.
- Resume local review drafts and see whether the PR changed since the draft started.
- Comment or submit reviews.
- Queue supported reviews when GitHub is unavailable.
- Refresh a pull request.
- Open the pull request on GitHub.

## Issues

Issues shows cached issues for the selected repository.

Use it to:

- Search and filter issues.
- Read issue conversations.
- Add issue comments.
- Refresh an issue.
- Open the issue on GitHub.

## Actions

Actions shows checks and workflow runs for the selected repository.

Use it to understand whether recent work is passing, failing, queued, or blocked.

## Branch Watch

Branch Watch highlights suspicious branch changes that may need care. It can warn about risky differences, missing safety references, merge queue evidence gaps, direct pushes, or release-sensitive branch situations.

Each finding explains what happened, why it matters, what evidence Fallback has, and which recovery path is available.

## Status

Status shows health information about GitHub and Fallback's ability to reach the services it depends on.

Use it when sync or writeback behavior seems unusual.

## Settings

Settings controls account connection, workspace behavior, sync timing, editor and terminal preferences, cache management, diagnostics, attention settings, command shortcuts, and Branch Watch preferences.
