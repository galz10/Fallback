# Command Palette And Shortcuts

The command palette is the fastest way to move around Fallback.

Use it when you know what you want but do not want to click through the sidebar.

## What You Can Do

The command palette can help you:

- Open views.
- Search watched repositories.
- Add a repository by `owner/name`.
- Search cached pull requests and issues.
- Search files and commits.
- Open a repository in a new Fallback window.
- Open a local repository folder in your editor, terminal, or Finder.
- Open a repository, issues page, or pull requests page on GitHub.
- Run common sync actions.
- Jump to local changes.
- Open recent operation records.

The available commands depend on the current view, selected repository, and whether the repository has a local folder.

## Default Shortcuts

Fallback includes default navigation shortcuts:

| Action        | Default shortcut |
| ------------- | ---------------- |
| Home          | `Ctrl+H`         |
| My Work       | `Ctrl+M`         |
| Notifications | `Ctrl+N`         |
| Code          | `Ctrl+C`         |
| Local Changes | `Ctrl+L`         |
| Pull requests | `Ctrl+P`         |
| Issues        | `Ctrl+I`         |
| Actions       | `Ctrl+A`         |
| Branch Watch  | `Ctrl+B`         |
| Settings      | `Ctrl+,`         |

You can change command palette shortcuts in Settings.

## Search Tips

Use simple terms first:

- A repository name, such as `fallback`.
- A repository owner, such as an organization name.
- A pull request or issue title.
- A pull request or issue number.
- A file name.
- A commit SHA or commit message.

If you paste a repository name in `owner/name` format, Fallback can offer to watch that repository.

## Repository-Scoped Search

When a repository is selected, the command palette can search within that repository. This keeps results focused when you are deep in one codebase.

When no repository is selected, it searches more broadly across watched repositories.
